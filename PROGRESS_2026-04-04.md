# 🚀 Azit 프로젝트 작업 일지 (2026-04-04)

---

## ✅ 주요 구현 사항

### 1. 엔티티 리팩토링 및 확장 (Step 1, 2)

- **Member 엔티티 개편:** 기존 `User`에서 `Member`로 이름을 변경하고, `age`, `bio`, `profile_url`, `role` 등 필드를 추가했습니다.
- **N:M 연관관계 설정:** `TechStack`(기술스택)과 `MemberStack`을 통해 유저가 여러 스택을 선택할 수 있게 모델링했습니다.
- **1:N 연관관계 추가:** `Career`(경력)와 `Availability`(가용시간) 엔티티를 생성하여 `Member`와 연결했습니다.

### 2. 프로필 관리 시스템 (Step 3, 4, 5)

- **ID 기반 프로필 접근:** `/profile/{id}` 경로를 통해 모든 사용자의 프로필을 조회할 수 있도록 통합 컨트롤러를 구현했습니다.
- **보안 및 권한 로직:**
  - 로그인한 사용자 본인만이 자신의 프로필을 수정할 수 있도록 `isOwnProfile` 플래그를 모델에 적용했습니다.
  - Spring Security를 연동하여 권한(`ROLE_USER`, `ROLE_ADMIN`)에 따른 접근 제어를 설정했습니다.
- **프로필 사진 업로드:** `MultipartFile`을 이용해 이미지를 `uploads/` 폴더에 물리적으로 저장하고, DB에는 웹 경로를 매핑하는 기능을 추가했습니다.

### 3. 관리자(Admin) 기능 및 기술 스택 연동 (Step 5 마무리)

- **전용 관리 페이지:** `/admin/stacks` 페이지를 생성하여 기술 스택을 직접 추가/삭제할 수 있는 환경을 만들었습니다.
- **아이콘 연동:** `Devicon` 등의 외부 URL 주소를 입력하여 기술 스택 배지 옆에 아이콘이 출력되도록 필드를 추가했습니다.

### 4. 동적 데이터 관리 기능 (Step 6)

- **Careers & Availability:**
  - Thymeleaf와 JavaScript를 결합하여 화면에서 입력 칸을 자유롭게 추가(`+ Add`)하거나 삭제(`X`)할 수 있는 동적 폼을 구현했습니다.
  - 배열 기반 파라미터를 서버에서 한꺼번에 처리하여 일괄 업데이트(Update/Delete-and-Insert) 로직을 완성했습니다.

## 🛠 사용된 기술 스택

- **Backend:** Java 25, Spring Boot 4.x, Spring Security 6 (Thymeleaf Extras)
- **Database:** MySQL 8.0 (Docker 기반), Hibernate/JPA
- **Frontend:** Bootstrap 5.3, Thymeleaf, JavaScript (Vanilla JS)

## 💡 수동 조작 가이드 (관리자 권한 부여)

DB에서 직접 권한을 승격시키고 싶을 때 아래 명령어를 터미널에 입력하세요:

```powershell
docker exec -it azit-mysql mysql -u root -p1234 -e "USE azit_db; UPDATE members SET role = 'ROLE_ADMIN' WHERE id = 1;"
```
